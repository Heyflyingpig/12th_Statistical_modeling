# Q1 Agent: 年度版住宅空置风险识别说明

## 1. 当前最终判断


1. 夜间灯光数据已经换成了正确的夜光数据。
2. 但其时间粒度是“年度夜光复制进季度表”，不是季度夜光。
3. 因此 Q1 的正式口径必须统一为 `2019-2023` 年度 RVRI。
4. 在这一口径下，当前方法是奏效的，可以继续采用方法二。

这里“奏效”的含义不是“在全市所有格网上，`rvri` 都必须与 `light` 强负相关”，而是：

在与住宅风险真正相关的建成区样本中，RVRI 能够稳定体现“建成强度高但夜间活力不足”的错配结构。

---

## 2. 当前 Q1 的正式任务

Q1 现在的正式任务应写成：

“基于 2019-2023 年格网尺度多源遥感特征，在年度尺度上构建住宅空置风险复合指数（RVRI），并验证其在建成区样本中的识别有效性与空间聚集特征。”

也就是说，Q1 不再是：

- 季度 RVRI 面板构建；
- 季度风险快照验证；
- 直接为季度状态转移提供底表。

Q1 现在是：

- 年度 RVRI 构建；
- 年度版内部逻辑验证；
- 年度版空间和外部效度验证；
- 为 Q2 和 Q3 提供年度风险状态。

---

## 3. 当前正式流程

### Step 1. 年度夜光写入

输入仍然是五个年度 CSV：

- `Shaoguan_RVRI_2019.csv`
- `Shaoguan_RVRI_2020.csv`
- `Shaoguan_RVRI_2021.csv`
- `Shaoguan_RVRI_2022.csv`
- `Shaoguan_RVRI_2023.csv`

通过：

- `Q1/01b_Inject_TIF_to_CSV.py`

将年度夜间灯光写入 CSV。

### Step 2. 将季度表统一解释为年度表

虽然原始 CSV 保留了 `time` 字段，但同一年内四个季度的 `light` 对同一格网完全一致，因此不能继续把这套数据解释为季度动态面板。

正确做法是：

1. 按 `grid_id + source_year` 聚合。
2. `ndbi` 取四季度均值。
3. `ndvi` 取四季度均值。
4. `light` 取四季度均值。

这一操作由：

- `Q1/rvri_pipeline.py`
- `Q1/02_RVRI_Synthesis.py`

负责完成。

### Step 3. 构造风险同向特征

在年度表上，使用三个正式风险特征：

1. `stock_pressure = z(ndbi)`
2. `eco_deficit = -z(ndvi)`
3. `mismatch_gap = z(ndbi) - z(light)`

这三个特征分别代表：

- 建成存量压力；
- 生态/环境退化；
- 建成-活力错配。

### Step 4. PCA 合成年度 RVRI

当前实跑结果：

- 校准年份：`2019-2023`
- 校准样本数：`396302`
- 第一主成分解释率：`74.00%`
- 主成分载荷：
  - `stock_pressure = 0.6412`
  - `eco_deficit = 0.5885`
  - `mismatch_gap = 0.4925`

说明当前 RVRI 的统计结构是稳定的。

## 4. 当前方法为什么判定为“奏效”

### 4.1 全样本不能直接作为是否奏效的唯一标准

在全市全部格网上：

- `corr_rvri_light = 0.0711`

如果只看这一个指标，会误以为方法失效。

但这个结论并不成立，因为全样本中包含大量山地、林地和低活跃非住宅格网，零值夜光会稀释住宅风险逻辑。

### 4.2 建成区子样本显示方法有效

对建成区样本重新检验后，结果明显改善：

#### 建成区子样本（NDBI 前 25%）

- `corr_rvri_light = -0.3514`
- `corr_rvri_mismatch = 0.5293`

#### 核心建成区（NDBI 前 10%）

- `corr_rvri_light = -0.6679`
- `corr_rvri_mismatch = 0.7195`

因此，当前方法应解释为：

“在住宅/建成区样本中，RVRI 能够有效刻画‘房多灯暗’的风险结构；全市全样本相关性偏弱主要来自非住宅稀疏格网的背景稀释。”

这就是当前选择“继续使用方法二”的主要依据。

当前后验验证分成两层：

1. `2019-2023` 全部年度样本用于检验方法本身是否有效；
2. `2023` 最新年度快照用于做空间格局、POI 外部效度和耦合协调的截面诊断。

之所以当前脚本默认只对 `2023` 做快照分析，不是因为前几年不能分析，而是因为：

1. Q1 当前的空间自相关、POI 和 CCDM 检验属于“截面结构诊断”；
2. 用最新年份最能代表当前风险格局，也更适合与论文中的结论图对应；
3. 方法有效性本身已经在 `2019-2023` 全年样本中完成检验，所以不是只用 `2023` 一个年份支持 Q1；
4. 如果后续 Q3 需要展示扩散过程，可以在同一套年度底表上补做 `2019-2023` 每年的 Moran's I 和 LISA 演化。


“Q1 采用 `2019-2023` 年度面板完成指数构建与方法有效性检验，并选取最新年份 `2023` 作为代表性截面，展示空间集聚和外部一致性特征。”

基于 `2023` 年度快照，当前结果如下：

### 6.1 模型元数据如何解读

当前模型元数据来自：

- `Q1/output/Q1_RVRI_Model_Metadata.json`

其中最重要的四类指标含义如下。

#### （1）`features`

当前正式合成特征为：

- `stock_pressure`
- `eco_deficit`
- `mismatch_gap`

它们分别对应：

- 建成存量压力；
- 生态/绿地不足；
- 建成强度与夜间活力不匹配。

这意味着当前 RVRI 不是单纯的“灯光弱”指数，而是一个同时考虑“建成程度、生态退化、建成-活力错配”的综合风险指数。

#### （2）`loadings`

当前载荷为：

- `stock_pressure = 0.6412`
- `eco_deficit = 0.5885`
- `mismatch_gap = 0.4925`

载荷越大，说明该特征对 RVRI 的贡献越大。

从当前结果看：

1. `stock_pressure` 权重最大，说明“建成存量高”仍是空置风险识别的核心基础；
2. `eco_deficit` 次之，说明低绿度、低生态环境质量区域与风险上升具有一致性；
3. `mismatch_gap` 仍然显著为正，说明“建成强但灯光弱”的错配结构确实被吸收到最终指数中。

这组结果符合我们的理论预期，即住宅空置风险首先发生在“已开发、已建设”的空间里，而不是简单地发生在“灯光低”的地方。

#### （3）`explained_variance_ratio`

- 第一主成分解释率 = `74.00%`

这表示 RVRI 的第一主成分已经能够解释三项核心风险特征的大部分共同变异。

在统计建模语境下，这个值说明：

1. 三个特征之间存在较强的一致结构；
2. 使用单一综合指数来代表“住宅空置风险”是可接受的；
3. 后续 Q2、Q3 使用 `rvri` 作为核心状态变量是有统计基础的。

如果解释率只有 30% 左右，我们就要怀疑这三个特征是否根本不在描述同一个潜在过程；但现在 `74.00%` 属于比较稳健的结果。

#### （4）`reference_rows`

- 校准样本数 = `396302`

这说明 PCA 不是在一个很小的截面样本上训练出来的，而是在 `2019-2023` 年度面板的大样本上校准完成的，因此稳定性相对更好。

### 6.2 方法有效性指标如何解读

当前综合结论是：

- `works_for_q1 = true`

这里的 `works_for_q1` 不是随意给出的布尔值，而是基于如下规则：

1. 在建成区样本中，`corr_rvri_light <= -0.30`
2. 在建成区样本中，`corr_rvri_mismatch >= 0.50`

也就是说，只有当 RVRI 在“建成区”这一真正 relevant 的样本里，同时满足：

- 与夜间灯光显著负相关；
- 与建成-活力错配显著正相关；

才会被判定为当前方法奏效。

#### （1）全样本结果

当前全样本结果为：

- `corr_rvri_light = 0.0711`
- `corr_rvri_mismatch = 0.6759`
- `corr_rvri_ndbi = 0.8917`
- `corr_rvri_ndvi = -0.8265`

这些值应这样理解：

1. `corr_rvri_light = 0.0711` 接近 0，说明把全市全部格网都混在一起时，RVRI 与灯光强弱并不形成稳定单调关系；
2. `corr_rvri_mismatch = 0.6759` 较高，说明 RVRI 与“房多灯暗”的错配逻辑整体一致；
3. `corr_rvri_ndbi = 0.8917` 很高，说明 RVRI 首先锚定在建成空间；
4. `corr_rvri_ndvi = -0.8265` 很强，说明低生态、低绿度区域风险更高。

这组结果表明：

RVRI 在全样本里更像是“建成空间风险强度”而不是“全市统一的夜光反函数”。

因此，全样本 `rvri-light` 相关弱，并不意味着模型失效，而是说明“非建成、低密、非住宅”格网会稀释夜光逻辑。

#### （2）建成区子样本结果

建成区子样本定义为 `NDBI` 前 `25%`：

- 样本量 `99076`
- 阈值 `ndbi_threshold = -0.1473`
- `corr_rvri_light = -0.3514`
- `corr_rvri_mismatch = 0.5293`

这一层结果比全样本更重要，因为住宅空置风险本来就应该在建成区中识别。

其含义是：

1. 在建成区内，灯光越弱，RVRI 越高，方向正确；
2. 在建成区内，建成-活力错配越大，RVRI 越高，方向正确；
3. 这表明当前指数并没有被“生态低”“建成高”完全绑死，而是开始真正识别出“活力不足”的结构。

#### （3）核心建成区结果

核心建成区定义为 `NDBI` 前 `10%`：

- 样本量 `39631`
- 阈值 `ndbi_threshold = -0.0706`
- `corr_rvri_light = -0.6679`
- `corr_rvri_mismatch = 0.7195`

这是当前方法最有说服力的一组结果。

它说明：

1. 当我们把样本进一步收缩到最典型的高建成强度区域后，“房多灯暗”逻辑显著增强；
2. RVRI 在核心建成区中已经能较强地刻画潜在空置风险；
3. 这说明当前 Q1 指标并不是“在所有地方都好”，而是“在应该好的地方确实好”。

这也是为什么当前方法可以继续保留，而不需要整体推翻重做。

### 6.3 住宅候选格网筛选方法如何解读

为避免把山体、裸地、远离聚落的非住宅格网误判为“住宅空置高风险”，当前空间自相关分析不再直接对全部格网计算，而是先构造“住宅候选格网”子样本。

当前筛选逻辑已经写入：

- `Q1/03_RVRI_Advanced_Validation.py`
- `Q1/output/Q1_Diagnostic_Full_Report.json` 中的 `residential_candidate_filter`

当前 `2023` 年的筛选结果为：

- `urban_quantile = 0.75`
- `urban_ndbi_threshold = -0.1407`
- `min_ndvi = 0.1`
- `snapshot_rows = 79260`
- `candidate_rows = 8405`
- `excluded_barren_or_remote_rows = 70855`
- `positive_light_candidate_rows = 6259`
- `neighbor_light_context_rows = 8379`
- `poi_context_rows = 79`

#### （1）建成区阈值如何筛选

建成区阈值采用：

- `NDBI` 的年度截面 `75%` 分位数

也就是：

1. 对每个年份单独计算该年快照的 `ndbi` 分布；
2. 取前 `25%` 的高建成格网作为建成区候选；
3. 只有 `ndbi >= urban_ndbi_threshold` 的格网，才有资格进入后续住宅候选筛选。

这样做的好处是：

1. 阈值随年度数据分布自适应变化；
2. 不依赖固定的人工常数；
3. 能把大部分低开发、低密度背景格网先排除掉。

就 `2023` 来说，这个阈值对应：

- `ndbi >= -0.1407`

这意味着只有建成特征相对更明显的格网，才会被当作潜在住宅风险候选。

#### （2）疑似裸地/山体排除阈值如何筛选

疑似裸地/山体排除阈值当前采用：

- `NDVI >= 0.1`

其含义是：

1. 如果格网的植被指数极低，接近裸地、裸岩、施工裸面或山体反射特征；
2. 即使它的 `ndbi` 较高，也不直接视为住宅候选格网；
3. 只有满足 `ndvi >= 0.1` 的格网，才允许进入住宅候选子样本。

这个阈值不是为了识别“绿化好的住宅区”，而是为了剔除那些：

- 看起来像高反射建设面；
- 但地理语义上更可能是裸地、山体、采石场、非住宅裸面；

的极端误识别格网。

#### （3）为什么还要加聚落上下文筛选

即使一个格网满足“建成强、不是裸地”，它仍然不一定是住宅活动空间。

因此当前还叠加了“聚落上下文”约束，要求至少满足以下之一：

1. 本格网 `light > 0`
2. 相邻格网存在正灯光
3. 本格网存在 POI

这样做的目的，是保留：

- 处在真实城镇环境中的暗住宅格网；

同时排掉：

- 孤立在山地或荒地中的高值格网。

#### （4）当前住宅候选筛选应如何表述

现在更准确的论文表述应为：

“为避免将山体裸地等非住宅高反射地表误识别为空置高风险区，本文在进行空间自相关诊断前，先依据建成度阈值、裸地排除阈值及聚落上下文约束构造住宅候选格网子样本，再在该子样本上计算 Moran's I 与局部聚集类型。”

### 6.4 空间自相关指标如何解读

- Global Moran's I = `0.6256`
- `p = 0.01`
- `HH` 聚集格网数 = `3210`
- 空间分析样本数 = `8405`

这三个指标应分别这样理解：

#### （1）Global Moran's I

Global Moran's I 用来判断空间相邻格网之间是否存在同向聚集。

- 若接近 `0`，表示空间随机；
- 若显著大于 `0`，表示高值与高值、低值与低值存在聚集；
- 若小于 `0`，表示高低交错分布。

当前 `0.6256` 仍然是较高的正值，说明在住宅候选格网内部，RVRI 不是随机散落的，而是具有明显的空间团聚性。

这对 Q3 非常关键，因为它意味着后续讨论“风险扩散”“相邻区域状态传导”是有空间基础的，而且这种基础来自更贴近住宅语义的子样本，而不是全图混算。

#### （2）`p = 0.01`

这里的 `p` 值来自随机置换检验，表示当前观察到的空间集聚强度不是随机巧合。

当前 `p = 0.01`，说明至少在当前置换设定下，空间自相关具有统计显著性。

#### （3）`HH` 聚集格网数

`HH` 表示高风险-高风险聚集，即一个高风险格网周边仍以高风险格网为主。

当前 `HH = 3210`，说明在住宅候选格网内部，高风险区不是零散点状，而是已经形成较为清晰的高风险聚集区域。

在论文里，这一指标可被解释为：

“住宅空置风险在空间上具有明显成片分布特征，存在潜在的区域扩散与锁定效应。”

与之前“全格网直接做 LISA”相比，这个结果更可信，因为它已经尽量排除了山体、裸地和远离聚落的非住宅高值区。

#### （4）结合 `2023` 韶关空间格局，当前结果应如何表述

如果进一步把 `2023` 年住宅候选格网中的 LISA 聚类结果按区县汇总，可以看到：

- 武江区、浈江区、曲江区的平均灯光更高，`HH` 占比相对更低；
- 南雄市、始兴县、翁源县等县域候选格网中，`HH` 占比相对更高；

其中 `2023` 年较有代表性的结果是：

- 武江区：`mean_light = 7.57`，`hh_share = 0.21`
- 浈江区：`mean_light = 4.68`，`hh_share = 0.21`
- 曲江区：`mean_light = 4.45`，`hh_share = 0.18`
- 南雄市：`mean_light = 1.97`，`hh_share = 0.88`
- 始兴县：`mean_light = 2.16`，`hh_share = 0.60`
- 翁源县：`mean_light = 2.35`，`hh_share = 0.45`

这说明在最新年份截面下，韶关中心城区整体活力仍然相对较强，而外围县域和县城周边更容易出现“建成存在但夜间活力不足”的高风险集聚现象。

因此，当前 `2023` 结果更适合写成：

“在剔除疑似山体裸地和远离聚落背景区之后，韶关市住宅候选格网的高风险集聚区更多分布于部分县域和外围城镇开发片区，而主城区整体表现出更强的低风险稳定性。该结果与韶关‘中心城区相对集聚、外围县域活力分化更明显’的城镇空间结构基本一致。”

这也是为什么当前阶段更应修正空间诊断口径，而不是贸然修改 RVRI 权重。

### 6.5 POI 外部效度指标如何解读

在建成区且存在 POI 的样本中：

- Spearman(`rvri`, `poi_count`) = `-0.2316`
- Pearson(`rvri`, `log(1+poi_count)`) = `-0.2083`

这里之所以同时报告 Spearman 和 Pearson，是因为两者回答的问题不同。

#### （1）Spearman 相关

Spearman 反映排序上的单调关系，对极端值不那么敏感。

当前 `-0.2316` 说明：

POI 越多的格网，RVRI 排名整体越靠低风险一侧。

#### （2）Pearson 相关

Pearson 更接近线性关系，因此这里对 `poi_count` 先做了 `log(1+poi_count)` 处理，减少超高 POI 区域的极端偏斜影响。

当前 `-0.2083` 说明：

就线性趋势而言，生活服务越活跃的地方，RVRI 也总体越低。

这两个值都不算特别高，但方向一致，这已经足以说明 RVRI 与“生活服务活跃度”存在外部一致性。

在论文表述中，更稳妥的说法应是：

“RVRI 与生活服务活跃度呈负向关系，表明居住与服务功能越活跃的区域，其疑似空置风险越低。”

### 6.6 CCDM 耦合校验指标如何解读

在建成区样本中：

- CCDM 均值 = `0.1711`
- `corr_rvri_vs_ccdm = -0.2593`

CCDM 这里衡量的是“建成强度”和“夜间活力”之间是否协调。

#### （1）`ccdm_mean`

- 当前均值 `0.1711`

这个值偏低，说明建成区整体上存在明显的“建设强度高于实际活力承载”的现象。

#### （2）`corr_rvri_vs_ccdm`

- 当前 `-0.2593`

负相关意味着：

协调度越高，风险越低；协调度越差，风险越高。

虽然该相关并不算特别强，但方向明确，说明 RVRI 并不是与协调发展相矛盾的指标，而是与“发展失衡”保持一致。

#### （3）CCDM 分级结果

当前分布为：

- `严重失调 = 13507`
- `轻度失调 = 4891`
- `勉强协调 = 1182`
- `优质协调 = 235`

这说明在建成区内部，大多数格网仍处于失调状态，而真正达到高协调的区域非常少。

如果把这一结果与高风险聚集区叠加，往往能更好地识别“建得快但活得弱”的区域。

### 6.7 年度风险水平走势如何解读

从年度主结果表看，`2019-2023` 每年的 `rvri` 均值大致为：

- `2019 = 0.4284`
- `2020 = 0.5200`
- `2021 = 0.4524`
- `2022 = 0.4205`
- `2023 = 0.3939`

这组结果至少说明两点：

1. 年度风险水平存在变化，不是一个静态常数；
2. `2020` 年显著偏高，而后逐步回落，说明样本中存在年度差异，后续 Q3 可以在年度转移中进一步检验这种波动是否体现为空间扩散或状态回落。

需要注意的是：

由于当前 `rvri` 是按年内最小-最大值缩放到 `[0,1]`，因此不同年份均值可比较的是“相对风险格局变化”，而不是绝对风险水平的线性差值。

如果只针对 `2023` 年最新快照，则当前可以把结论概括为：

1. 年度 RVRI 均值为 `0.3939`，说明 `2023` 年总体风险水平低于 `2020` 高点；
2. 但住宅候选格网内部仍存在显著空间集聚，说明风险并未消失，而是以更局部、更片区化的方式存在；
3. 因此 `2023` 年更适合作为“高风险片区识别”和“空间扩散边界判断”的代表年份。

## 7. 当前每一张图应如何解读

### 7.1 `Q1_RVRI_Scientific_Check.png`

这张图是“建成区样本中夜间灯光与 RVRI 的散点检验图”。

- 横轴：`Nighttime Light`
- 纵轴：`RVRI`
- 颜色：`NDBI`

它的作用不是证明所有点都严格落在一条直线上，而是观察在建成区中是否总体存在：

1. 灯光偏弱时，RVRI 偏高；
2. 高建成强度样本中更容易出现高风险点。

如果图中整体呈“从左上到右下”的趋势，就说明“房多灯暗”逻辑基本成立。

在论文里，这张图可作为：

“内部一致性检验图”，说明指数与核心理论假设方向一致。

### 7.2 `Q1_LISA_Map.png`

这张图是“住宅候选格网子样本”上的年度局部空间聚集图，不再是对全部格网直接作图。

当前图面包含三层信息：

1. 全市格网框架：浅灰色底图，用于保留韶关市整体空间轮廓；
2. 展示掩膜区域：浅卡其色格网，表示这些格网具备较宽松的建成语义背景，用于保证图面连续性和可读性；
3. 被排除区域：深灰色格网，表示这些格网既未进入严格分析掩膜，也未进入宽松展示掩膜；
4. LISA 聚类层：仅对严格住宅候选格网着色，展示 `HH/LL/HL/LH` 四类聚集关系。

- 红色 `HH`：高风险周边仍是高风险
- 蓝色 `LL`：低风险周边仍是低风险
- 粉色 `HL`：高风险被低风险包围
- 浅蓝 `LH`：低风险被高风险包围

这张图最重要的意义不是“颜色好看”，而是识别住宅候选空间中的结构类型：

1. `HH` 连片区可解释为住宅候选空间中的高风险集聚区；
2. `HL/LH` 边界带可解释为潜在转化带或过渡带；
3. `LL` 连片区可解释为风险较稳定、活力较强区域。

对 Q3 来说，这张图尤其重要，因为它直接提示未来最值得关注的空间扩散边界。

需要特别强调的是：

红色 `HH` 现在应解释为：

“住宅候选格网中的高风险聚集区”

而不应再解释为：

“全图任意位置上的高风险聚集区”。

因此，如果后续图上仍出现明显山地区块，应优先检查候选格网掩膜是否还需要进一步收紧，而不是直接把该区域解释为“有大量空置住宅”。

灰色区域则应解释为：

“未参与住宅空置空间聚类分析的非候选区域，包括疑似山体裸地、非建成背景区或缺乏聚落上下文支撑的格网。”

浅卡其色区域则应解释为：

“仅用于地图展示的宽松建成背景区。这部分区域保留在图中，是为了增强地图完整性和空间可读性，但并不直接参与 Moran's I 与 LISA 聚类统计。”

按照当前 `2023` 年结果，这张图最适合支撑的答案不是：

“韶关山区本身存在大量住宅空置高风险区”

而是：

“在保留全市地图框架的前提下，真正进入住宅空置空间聚类分析的高风险区，主要集中在具备建成和聚落上下文的候选格网之中；山区和被排除背景区仅作为地理参照，不参与高风险聚类解释。”

### 7.3 `Q1_POI_Validation_Enhanced.png`

这张图是 `log(1 + POI count)` 与 `RVRI` 的回归散点图。

- 横轴：`log(1 + POI count)`
- 纵轴：`RVRI`
- 红线：趋势线

这张图的意义是：

如果回归线整体向下，说明生活服务越活跃，风险越低。

它不能直接证明“POI 少就一定空置”，但能说明：

RVRI 与城市功能活跃度在方向上保持一致，因此具有一定外部效度。

### 7.4 `Q1_Step6_KDE_Coupling_Validation.png`

这张图是夜间灯光与 `NDBI` 的六边形密度图。

- 横轴：`Nighttime Light`
- 纵轴：`NDBI`
- 颜色深浅：样本密度

这张图的意义在于观察：

1. 高 `NDBI` 是否都对应高灯光；
2. 是否存在明显“高建成、低灯光”的密集区域；
3. 建成与活力是否存在系统性失衡。

如果高密度区域集中在“较高 NDBI、较低 light”位置，就说明样本中确实存在建成-活力错配的结构基础。

## 8. 当前正式输出文件

### 8.1 年度主结果

- `Q1/output/Shaoguan_RVRI_Q1_Validated.csv`

这是 Q1 的核心底表。后续 Q2 和 Q3 都应优先读取这个文件。

主要字段可理解为：

- `rvri`：按年标准化后的风险得分
- `risk_state`：按年三分位划分后的风险状态
- `time_grain = annual`：明确该数据是年度版
- `quality_flag = annual_ntl_replicated`：说明当前夜光来源本质是年度夜光复制到季度表后再聚合而成

### 8.2 模型元数据

- `Q1/output/Q1_RVRI_Model_Metadata.json`

这个文件记录“Q1 是怎么被合成出来的”，后续论文写作与复现实验都应引用它。

### 8.3 年度数据审计

- `Q1/output/Q1_Data_Quality_Audit.json`

这个文件主要用于说明数据质量和时间粒度问题。

其中最值得关注的字段是：

- `temporal_grain`
- `quarter_constant_share`
- `positive_light_rate`
- `usable_for_calibration`

它们的含义分别是：

1. `temporal_grain`：当前年份夜光是季度级还是年度复制级；
2. `quarter_constant_share`：同一格网四季度灯光完全相同的比例；
3. `positive_light_rate`：夜光大于零的样本比例；
4. `usable_for_calibration`：该年份是否能够作为 RVRI 校准样本。

### 8.4 年度综合诊断

- `Q1/output/Q1_Diagnostic_Full_Report.json`
- `Q1/output/validation_report.json`

其中：

1. `Q1_Diagnostic_Full_Report.json` 适合做论文结果与方法解释；
2. `validation_report.json` 适合快速查看关键指标总览。
